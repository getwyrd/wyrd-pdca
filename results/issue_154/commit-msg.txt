chore: M2.7 follow-up build/bench/repo hygiene

A grouped set of low-severity hardening items surfaced reviewing the M2.7
Tier-2 integration/throughput work (#117). None changes the data path or
fixes a live bug; each is a small, independent cleanup that removes a way
the build, the test harness, or the docs could mislead later.

- Pin the d-server build image to `rust:1.96.0-bookworm` (was the floating
  `1.96-bookworm`) and build it `--locked`, so the container compiles with
  the same toolchain and lockfile the workspace is gated on instead of
  silently re-resolving to a newer patch or newer deps.
- Make the Tier-2 integration teardown panic-safe: a panicking test (as
  opposed to a non-zero `cargo test` exit) used to unwind past finalization,
  leaking the Docker cluster and capturing no diagnostics. The body now runs
  under a wrapper that captures logs and tears the cluster down on every
  exit path, then resumes the panic.
- Warn instead of silently clamping a rejected `WYRD_DSERVER_COUNT`: a
  typo'd `WYRD_DSERVER_COUNT=1` previously ran a 9-server cluster with no
  signal; an unusable value now falls back to the default and says so.
- Correct the bench `Cluster` doc, which claimed dropping the cluster shuts
  the servers down — a dropped tokio `JoinHandle` detaches the task, it does
  not abort it.
- Add `.github/dependabot.yml` (cargo + github-actions, weekly) so a new
  RUSTSEC advisory arrives as a tracked update PR rather than a surprise
  `cargo deny check` failure.
- Drop the inert `results/` line from `.dockerignore` (nothing in the repo
  produces that directory).
- Add a one-paragraph note to the architecture quality-risks glossary
  disambiguating its Tier 0-3 realism ladder from the code/CI "Tier-1"/
  "Tier-2" test-taxonomy labels, which collide on the bare word "Tier 2".

The panic-safe teardown is stacked on the #150 teardown rework
(capture-logs-before-teardown) and must land after it; the wrapper delegates
to #150's finalizer rather than replacing it so the two compose. Do not
co-schedule #150 and #154 in one concurrent wave.
