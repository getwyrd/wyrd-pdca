docs(readme): add a Development & testing on-ramp

The README's only build/test guidance was the standard `cargo build` /
`cargo test` flow, so a newcomer had no way to discover the real
workflow without reading source or ADRs. The merge gate (`cargo xtask
ci`), the Tier-2 container tier (`cargo xtask integration`), `bench`,
`dst`, and `conformance` were all undocumented; plain `cargo test`
silently skips the `#[ignore]`d Tier-2 test; the `WYRD_DSERVER_COUNT`
knob (default 9, minimum 2) was invisible; and the self-contained
`wyrd demo` round-trip went unmentioned even though the README said only
"not yet deployable".

Add a "Development & testing" section that lists the `cargo xtask` entry
points, states the Docker + Compose-plugin prerequisite for the
integration tier, documents `WYRD_DSERVER_COUNT`, and gives a one-line
"Try it" via `cargo run -p wyrd-server --bin wyrd -- demo`.

The root README sits in no gate (`cargo xtask ci` is Rust-only; the docs
linter covers only `docs/`), so the on-ramp can drift from the code.
A new `xtask/tests/readme_dev_section.rs` cross-checks each documented
command against the actual dispatch in `xtask/src/main.rs` and
`crates/server/src/cli.rs`, so it fails if the README names a command
that does not exist or one is renamed in source without an update.

Fixes #152
