xtask: capture Tier-2 container logs before teardown on failure

The nightly Tier-2 container job is the only place a Tier-2 regression
surfaces (post-merge, non-required by design, ADR-0009), but a failing
run left nothing to diagnose it: the integration runner tore the cluster
down before propagating the error, so by the time the failure surfaced
the containers, their logs and volumes were already destroyed, and the
workflow never captured `docker compose logs`.

Restore the operability invariant that a failed run captures its
diagnostic evidence before destroying the environment that produced it.
On failure the runner now captures container logs first and only then
tears the cluster down; teardown still always runs, so a run never leaks
containers. The logs are echoed to the job log for at-a-glance triage
and persisted under target/tier2-logs/.

The nightly workflow gains a `timeout-minutes` bound so a hung cold
build fails fast instead of drifting toward the 6h default, and an
`if: failure()` step that uploads the captured diagnostics as an
artifact so they survive job-log truncation for post-merge triage.

Scope is the nightly Tier-2 job's operability only; the gating model is
unchanged (Tier-2 stays nightly and non-required, per ADR-0009).

Fixes #150
