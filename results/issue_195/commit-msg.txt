test(xtask): build the Tier-1 disk-fault harness

The M3 verification campaign promised a Tier-1 disk-fault leg that drives the
custodian repair path against real block-layer faults, but only inert
scaffolding shipped: the disk-fault runner shelled out to an external command
defined nowhere in the tree, so the promised coverage was effectively absent.

Build the Tier-1 disk-fault harness as real, in-repo Rust, mirroring the
existing Tier-2 container harness. The disk-fault runner now drives an in-repo
scenario that opens a real FsChunkStore on a device-mapper-backed device,
corrupts a stored fragment, and runs the production scrub and reconstruction
path over it — asserting the faulted chunk is driven back to full redundancy
with no read errors propagated to the caller during repair.

The host-independent orchestration (the device-mapper table plan and the
post-repair campaign verdict, covering both the scrub and reconstruction legs)
is unit-tested in the unprivileged gate, so the harness is exercised on every
run rather than only when privileged hardware is present. The real-device
scenario is compiled and type-checked by the same gate, and its privileged run
against a dm-error device is exercised off-gate by a new nightly workflow.

The reconstruction leg relies on the block-layer read-around delivered in #251;
this change builds on that and adds no production custodian, scrub, or
reconstruction code.

Fixes #195
